    View Source Bonfire.Common.ConfigModules (bonfire v0.3.1-classic-beta.9)

    A Global cache of known config modules to be queried by associated schema, or vice versa.

    Use of the ConfigModules Service requires:

    1. Exporting config_module/0 in relevant modules, returning a Module or otp_app atom
    2. To populate :bonfire, :config_modules_search_path in config the list of OTP applications where config_modules are declared.
    3. Start the Bonfire.Common.ConfigModules application before querying.
    4. OTP 21.2 or greater, though we recommend using the most recent release available.

    While this module is a GenServer, it is only responsible for setup of the cache and then exits with :ignore having done so. It is not recommended to restart the service as this will lead to a stop the world garbage collection of all processes and the copying of the entire cache to each process that has queried it since its last local garbage collection.
